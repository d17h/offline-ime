### 4.2 内存状态分级

| 级别 | 内存阈值 | 状态描述 | 处理策略 |
|------|----------|----------|----------|
| **NORMAL** | < 60MB | 正常 | 正常运行，无限制 |
| **WARNING** | 60-80MB | 警告 | 降低动画质量、清理Bitmap缓存 |
| **CRITICAL** | 80-95MB | 严重 | 关闭动画、释放符号缓存、减少候选词 |
| **EMERGENCY** | > 95MB | 紧急 | 最简UI、关闭手写/语音引擎、强制GC |

### 4.3 自动降级策略

```kotlin
object MemoryThresholds {
    const val LIGHT_THRESHOLD_MB = 60
    const val MODERATE_THRESHOLD_MB = 80
    const val SEVERE_THRESHOLD_MB = 95
    const val EMERGENCY_THRESHOLD_MB = 110
}

class AutoDegradeManager(private val context: Context) {
    
    fun onMemoryLevelChanged(level: MemoryLevel) {
        when (level) {
            MemoryLevel.NORMAL -> restoreNormal()
            MemoryLevel.WARNING -> applyLightDegrade()
            MemoryLevel.CRITICAL -> applyModerateDegrade()
            MemoryLevel.EMERGENCY -> applySevereDegrade()
        }
    }
    
    private fun applyLightDegrade() {
        // 轻度降级：降低动画质量
        AnimationManager.setQuality(AnimationQuality.LOW)
        BitmapPool.getInstance().trimToSize(4 * 1024 * 1024)
    }
    
    private fun applyModerateDegrade() {
        // 中度降级：关闭动画、释放缓存
        AnimationManager.disableAll()
        SymbolManager.getInstance().clearCache()
        CandidateManager.setMaxCandidates(5)
        BitmapPool.getInstance().clear()
    }
    
    private fun applySevereDegrade() {
        // 严重降级：简化UI、关闭引擎
        KeyboardView.setSimplifiedMode(true)
        HandwritingEngine.release()
        VoiceEngine.release()
        ClipboardManager.setMaxItems(50)
    }
}
```

### 4.4 内存监控代码

```kotlin
class MemoryMonitor private constructor() {
    
    private val handler = Handler(Looper.getMainLooper())
    private val monitoringInterval = 5000L // 5秒
    private var isMonitoring = false
    
    private val monitoringRunnable = object : Runnable {
        override fun run() {
            checkMemoryStatus()
            if (isMonitoring) {
                handler.postDelayed(this, monitoringInterval)
            }
        }
    }
    
    fun startMonitoring() {
        isMonitoring = true
        handler.post(monitoringRunnable)
    }
    
    private fun checkMemoryStatus() {
        val runtime = Runtime.getRuntime()
        val usedMemory = (runtime.totalMemory() - runtime.freeMemory()) / (1024 * 1024)
        val maxMemory = runtime.maxMemory() / (1024 * 1024)
        
        val level = when {
            usedMemory > 95 -> MemoryLevel.EMERGENCY
            usedMemory > 80 -> MemoryLevel.CRITICAL
            usedMemory > 60 -> MemoryLevel.WARNING
            else -> MemoryLevel.NORMAL
        }
        
        AutoDegradeManager.getInstance().onMemoryLevelChanged(level)
    }
}
```

## 六、技术实现要点

### 6.1 词库引擎

- **存储**：SQLite FTS4虚拟表，支持全文检索
- **压缩**：拼音音节9bit编码，节省50%空间
- **索引**：多级索引（前缀+复合+T9数字）
- **响应时间**：< 50ms

### 6.2 手写识别

- **模型**：HandNet-Lite（MobileNetV3变体）
- **大小**：INT8量化后2.5MB
- **准确率**：96.5%
- **识别时间**：45ms

### 6.3 语音输入

- **引擎**：Vosk离线语音识别
- **模型**：vosk-model-small-cn-0.22（42MB）
- **准确率**：90-93%
- **内存增量**：26-34MB

### 6.4 符号系统

- **基础符号**：硬编码，12KB
- **Emoji**：GZIP压缩JSON，469KB
- **懒加载**：分类按需加载
- **缓存**：LRU最近使用20个

### 6.5 剪贴板

- **数据库**：Room + SQLCipher加密
- **容量**：最多200条
- **拆词**：12种类型正则规则
- **清理**：30天过期自动清理
